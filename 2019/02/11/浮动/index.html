    CSS中的流
    2019/02/11 前端 CSS

    简单总结一下’破坏流’和’保护流’的 CSS 属性。

    魔鬼属性 float

    1. float 的本质与特性
      浮动的本质是为了实现文字环绕效果
      浮动的特性:

      • 包裹性
      • 块状化并格式上下文
      • 破坏文档流
      • 没有任何的 margin 合并

    2. float 的作用机制
      让父元素的高度塌陷

    天然克星 clear
    1. clear 属性
      能够清除高度塌陷
      元素盒子的边不能和前面的浮动元素相邻
      只对块级元素有效
    /* 10个<li>元素排列 */
li {
  width: 20px;
  height: 20px;
  margin: 5px;
  float: left;
}
li:nth-of-type(3) {
  clear: both;
}
/* 会排成两行，第一个包含两个方块，剩余方块全在第二行。 */
/* 因为clear属性让自身不能和前面的浮动元素相邻。*/

    属性值: - none: 默认值，左右浮动来就来 - left: 左侧抗浮动 - right: 右侧抗浮动 - both: 两侧抗浮动

    BFC

    1. 定义
      BFC(Block Formatting Context): 块级格式化上下文

    2. 特性:

      • 内部的 Box 会在垂直方向，一个接一个地放置。
      • Box 垂直方向的距离由 margin 决定。属于同一个 BFC 的两个相邻 Box 的 margin 会发生重叠
      • 每个元素的 margin box 的左边， 与包含块 border box 的左边相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。
      • BFC 的区域不会与 float box 重叠。
      • BFC 就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
      • 计算 BFC 的高度时，浮动元素也参与计算

    3. 触发规则:

      • html 根元素
      • float 不为 none
      • overflow 为 auto/scroll/hidden
      • display 为 table-cell/table-caption/inline-block
      • position 不为 relative/static
