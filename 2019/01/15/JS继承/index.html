关于类的继承

2019-01-15
  • 前端
  • JS
  1. 构造函数继承
  2. 原型链继承
  3. 组合继承
  4. 组合继承优化(寄生)

在JS中并不存在类这一说法，class只是语法糖，本质上还是函数。

class Person(){}
Person instanceof Function   //true

今天就逐步优化一下类继承的方式。

构造函数继承

function Parent() {
  this.name = "Parent";
}
Parent.prototype.say = function () {
  console.log(this.name);
};
function Child() {
  Parent.call(this);    //可以向父类传递参数
  this.name = "Child";
}
new Child().say();    //not a function

特点: 用call()、apply()将父类构造函数引入子类函数

缺点: 只继承了父类构造函数的属性，没有继承父类原型的属性

原型链继承

function Parent() {
  this.name = "Parent";
  this.list = [1, 2, 3]
}
function Child() {
  this.name = "Child";
}
Child.prototype = new Parent();
let c1 = new Child();
let c2 = new Child();
console.log(c1.list);   //[1,2,3]
c2.list.push(5);
console.log(c1.list);   //[1,2,3,5]

特点: 让新实例的原型等于父类的实例

缺点: 所有子类共享父类实例的属性(原型上的属性是共享的，一个实例修改了原型属性，另一个实例的原型属性也会被修改)

组合继承

常见的继承方式之一

function Parent() {
  this.name = "Parent"
}
Parent.prototype.say = function () {
  console.log(this.name);
};
function Child() {
  Parent.call(this)   //构造函数继承，第一次调用Parent()
  this.name = "child"
}
Child.prototype = new Parent()   //原型链继承，第二次调用Parent()

特点: 结合使用构造函数和原型链的特点

缺点: 在继承父类函数的时候调用了父类构造函数，子类的原型上多了父类属性。两次调用构造函数。

组合继承优化(寄生)

function Parent() {
  this.name = "Parent"
}
Parent.prototype.say = function () {
  console.log(this.name);
};
function Child() {
  Parent.call(this)   //构造函数继承
  this.name = "child"
}
Child.prototype = Object.create(Parent.prototype)   //寄生
Child.prototype.constructor = Child
new Child() instanceof Parent    //true
new Child().constructor     //Child
